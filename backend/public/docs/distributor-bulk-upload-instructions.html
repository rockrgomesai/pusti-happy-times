📊 Distributor Bulk Upload Instructions

1. Prerequisites

⚠️ IMPORTANT: Before uploading distributors, ensure the following master data exists in the system:
  • Territories: All DB Points (Zone → Region → Area → DB Point) must be created
  • Facilities: All delivery depots/factories must be created
  • Navigate to Master → Territories and Master → Facilities to verify

2. Upload Process

  1. Click "Download CSV Template" button on the Distributors page
  2. Open the template in Excel, Google Sheets, or any CSV editor
  3. Fill in distributor data following the column specifications below
  4. Save the file as CSV format (UTF-8 encoding recommended)
  5. Click "Upload CSV" button and select your file
  6. Review the upload results - any errors will be displayed with row numbers

3. CSV Column Specifications

Required Columns *

Column Description Valid Values / Format
name Distributor name (must be unique) 3-160 characters, no duplicates
db_point_name DB Point territory name (exact match) Must exist in Territories with type "DB Point"
product_segments Product categories BIS, BEV, or BIS,BEV (comma-separated)
distributor_type Distributor classification Commission Distributor, General Distributor, Special Distributor, Spot Distributor, Super Distributor
delivery_depot_name Delivery facility name (exact match) Must exist in Facilities
unit Order unit CTN (carton) or PCS (pieces)

4. Important Notes

📝 Tips for Success:
  • Exact Name Matching: Territory and facility names must match exactly (case-sensitive)
  • ⚠️ CRITICAL - Phone Numbers in Excel: Always prefix mobile numbers with single quote ('8801712345678) to prevent Excel from converting to scientific notation (8.80171E+12). The quote forces text format and preserves leading zeros.
  • Phone Format: Bangladesh format: '8801XXXXXXXXX or '01XXXXXXXXX
  • Multiple Phones: Separate with commas: '01712345678, '01812345678
  • Date Format: Always use YYYY-MM-DD format (e.g., 2020-01-10)
  • Decimals: Use dot (.) for decimals, not comma: 500000.00
  • Empty Fields: Leave optional fields blank if not applicable

5. Error Handling

  • If validation fails, no distributors will be created (atomic transaction)
  • Review error messages carefully - they include row numbers and specific issues
  • Fix all errors in your CSV and re-upload the entire file
  • Common errors: duplicate names, invalid territory/facility names, wrong phone format, missing required fields
🆘 Need Help? Contact your system administrator or refer to the Master Data modules to verify territory and facility names before uploading.
